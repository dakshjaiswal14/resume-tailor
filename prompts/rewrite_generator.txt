CRITICAL MINDSET: LATEX IS SOURCE CODE

- Treat every LaTeX command, environment, and brace as SOURCE CODE — NOT as text.
- Do NOT rewrite the document. Do NOT regenerate the document. Do NOT restructure.
- Modify ONLY the sentence(s) inside bullet points. Leave everything else untouched.
- Every LaTeX command (\\textbf, \\textit, \\href, \\item, \\begin, \\end, etc.) is IMMUTABLE unless absolutely necessary for the specific text change.
- Edit only the HUMAN-READABLE TEXT inside commands. Never edit the commands themselves.

---

COMPILER RULES — ANY violation means the output is INVALID:

1. NEVER emit these invalid sequences:
   - \\\\\\% (double backslash before percent) — a % on its own is a comment; escape as \\\\
   - unmatched braces: every {{ MUST have a matching }}
   - malformed \\textbf{{}} or \\textit{{}}: must be exactly \\command{{text}}
   - malformed \\href{{}}: must have exactly two braced arguments — \\href{{url}}{{display text}}
   - malformed \\item: must be inside an itemize or enumerate environment
   - malformed environments: every \\begin{{...}} MUST have a matching \\end{{...}}

2. DO NOT TOUCH document structure:
   - Never modify \\vspace, \\hspace, \\hfill, tabular, tabular*, margins, title formatting
   - Never add fixed spacing commands
   - Preserve all existing indentation, line breaks, and formatting exactly

3. BRACE DISCIPLINE:
   - Count opening and closing braces before outputting
   - Every {{ has exactly one matching }}
   - \\textbf{{keyword}} — correct. \\textbf\\{{keyword\\}} — INVALID.
   - \\textbf{{nested \\textit{{italic}} text}} — correct (properly nested)

4. FORMATTING convention:
   - Bold: \\textbf{{text}}
   - Italic: \\textit{{text}}
   - NEVER use Markdown: **text**, *text*, __text__

5. ESCAPING:
   - A literal % sign in text MUST be written as \\%
   - A % at line start IS a LaTeX comment — leave comments untouched
   - Do NOT put a backslash before a % that is already a comment
